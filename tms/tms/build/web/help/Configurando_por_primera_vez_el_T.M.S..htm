Configurando por primera vez el TMS

El T.M.S. maneja 3 tipos de perfiles hay un único y exclusivo ADMIN (éste nombre no puede ser modificado), existe un rol de Administrador de Proyectos y Usuario  Cliente..

El ADMIN administra todas las opciones que componen el sistema. Para hacer uso de esta opción, debe conectarse como el administrador del sistema, es el que tiene el status ADMIN, para lo que Usted requiere tener un nombre de usuario que por lo general es ADMIN y un password, de lo contrario no podrá incluir datos o modificar los existentes. Estas claves las debe solicitar a su distribuidor TMS más cercano.

ADMIN: A éste menú solamente puede tener acceso las personas definidas como Administrador.

 Una vez conectado como usuario ADMIN y seleccionar la opción ADMIN  le aparecerá esta pantalla.

 

Las opciones de configuración las debe seguir en el orden que se las presentamos a continuación, ya que en ciertos puntos no podrá continuar sin antes haber llenado información que será requerida en otras opciones. Estas opciones de mantenimiento son las siguientes.

Administración de Usuarios    [Volver]

Se utiliza los botones de ver, incluir, eliminar o modificar usuarios del sistema, los cuales pueden tener distintos privilegios dependiendo del tipo de perfil escogido. Esta es una de las partes más importantes del sistema T.M.S., permite crea un grupo de usuarios que pueden hacer uso del sistema. Para cada uno de los miembros se define en esta pantalla una serie de elementos y algunos otros parámetros adicionales.

Descripción de campos

Usuario: Nombre que identifica al usuario dentro del sistema. Por lo general se utiliza la  inicial del nombre y el primer apellido completo.

Nombre: Nombre y apellidos del usuario

Título: Corresponde al puesto que desempeña el usuario. Por ejemplo: Analista, Programador, Director de Proyectos, etc.  Este nombre depende de la estructuración interna que tiene la compañía. También se definen en este campo los usuarios de la parte administrativa; por ejemplo: las personas encargadas de dar seguimiento a los contratos.

Organización: Corresponde al nombre de la empresa a la que va a pertenecer el miembro del proyecto.

Correo electrónico: Dirección electrónica del usuario donde desea recibir su correspondencia.

Teléfono trabajo: Código de área y número de teléfono del usuario donde se puede localizar.

Teléfono casa: Código de área y número de teléfono de la casa de habitación del usuario donde puede ser localizado.

Teléfono celular: Número de teléfono del celular o móvil del usuario.

Fax: Número de fax o facsímil donde recibe información el usuario.

Costo: Costo que tiene el proyecto para llevarse a cabo.

Comentarios: Este campo se utiliza para anotar alguna observación o comentario que se estime conveniente.

Template: Como desea ver el formato de la pantalla.

Verificar agenda calendario: Si el check de selección se encuentra marcado indica que va a ?????????.

Especifique la clave de acceso

Contraseña: Digite la clave de acceso o password con la que va a ingresar al sistema ese usuario.

Redigite la clave de acceso: Redigite la clave de acceso o password con la que desea ingresar al sistema.

Permisos del Usuario

Acceso de Administración de Proyectos: Este usuario tiene acceso a todas las opciones del T.M.S., sin ninguna restricción.

Acceso de Usuario: El miembro del equipo solo tiene acceso como Usuario del T.M.S.

Información del Sistema   [Volver]

Esta pantalla es solamente de información, los datos no pueden ser modificados. Aparece la información correspondiente a las propiedades del sistema.

Detalles de la Empresa    [Volver]

En esta pantalla se da mantenimiento a la información detallada de la empresa que es la dueña del producto MBACASE adquirido.

Nombre: Nombre de la empresa dueña del sistema.

Dirección: Dirección física donde se encuentra ubicada la empresa.

Apartado postal: Número de apartado postal que tiene la compañía para recibir la correspondencia.

País: Seleccione el país al que pertenece la empresa.  Estos países deben estar previamente definidos en el menú Admin.

Teléfono: Código de área y número de teléfono que tiene la compañía dueña del producto.

Web Site: Indique el nombre del web site o sitio en caso de que la compañía posea su propia página web en la internet.

Correo Electrónico: Dirección del correo electrónico donde recibe su correspondencia virtual.

Comentarios: Campo para anotar algún comentarios.

Indexar Información    [Volver]

Toda la información almacenada dentro del T.M.S. puede ser consultada mediante la opción BUSCAR, solo debe digitar una palabra CLAVE y el T.M.S. le indicará todos los lugares donde la misma aparece.  Este proceso se debe correr periódicamente de acuerdo a la cantidad de información que se va ingresando en el sistema, lo que hace es ordenar los índices de la base de datos que maneja el sistema.

 Recuerde: El proceso de indexación consume tiempo de CPU y puede demorar las otras opciones de los usuarios, se recomienda su ejecución por las noches o mejor si se deja como un proceso que se activa en forma automática.

 Recuerde: Utilice ésta opción cuando nota que el sistema está trabajando en forma lenta.

Tipos de Tarea    [Volver]

Pantalla para dar mantenimiento a las diferentes tareas que deben ser ejecutadas para un determinado proyecto.

Agregar Tipos de Tarea    

Descripción: Descripción del tipo de tarea.

Prefijo: Para cada uno de los tipo de tareas se puede definir un prefijo de forma tal que cuando se crea una nueva tarea en el sistema, ese prefijo se coloca en forma automática para identificarlo con un número que el sistema determina en forma automática que es lo que se conoce como consecutivo.  Por ejemplo Prefijo  ADM. Tareas de administración. y se crea una tarea tipo administrativo el sistema crea una tarea ADM-1 y el nombre asignado al usuario.

Consecutivo: Número consecutivo que identifica el tipo de tarea asignado por el sistema.

Usar Prefijo: Si el check de selección se encuentra marcado indica que el tipo de tarea debe utilizar el prefijo asignado.
