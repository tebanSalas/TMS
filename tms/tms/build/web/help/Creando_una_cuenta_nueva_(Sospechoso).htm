Trabajando con los proyectos del TMS

En esta opción no solo se definen proyectos, también se da mantenimiento a las Tareas, Costos, Riesgos, Tickets, Equipo y Contenido Enlazado.

El proyecto se está ejecutando actualmente.

El proyecto ha sido finalizado, se concluyeron las tarea asignadas al mismo.

Para cada uno de los proyectos Usted puede ver los siguientes datos:

: Si el check de selección se encuentra marcado indica que el usuario va a trabajar con ese proyecto.

Proyecto: Nombre del proyecto.

Estado: Estado en el que se encuentra el proyecto en el momento de la consulta.

Dueño: Dueño del proyecto.

Fecha inicio: Fecha a partir de la cual se dió inicio al proyecto.

Fecha final: Fecha en la que el proyecto debe finalizar.

Atrasadas: Muestra por línea de detalle las tareas que se encuentran atrasadas en el proyecto.

Asignado: Muestra por línea de detalle las tareas que se encuentran asignadas en el proyecto.

Sin asignar: Muestra por línea de detalle las tareas que aún no han sido asignadas en el proyecto.

Tickets: Son las discusiones que se abren para  un tema en particular del proyecto.

Si usted da un click sobre un proyecto determinado el sistema le permite ver en forma detallada mostrando la información general del mismo.

Para crear un proyecto Usted debe dar click al botón Crear y el sistema le abre la siguiente pantalla.

Los datos a ingresar son los siguientes:

Nombre: Nombre del proyecto.

Prioridad: Tipo de prioridad que tiene el proyecto dentro de la empresa.

Descripción: Breve descripción del proyecto. Este campo no es requerido por lo tanto puede indicarle N/A No Aplica.

Dirección: Dirección física donde se encuentra ubicada la empresa.

Dueño: Seleccione el dueño o director de proyecto que es la persona responsable de velar por el cumplimento del proyecto.

Organización: Seleccione la organización o empresa a la que pertenece el proyecto.

Estado: Estado en el que se encuentra el proyecto.  Siempre que se crea un proyecto nuevo tiene el estado será No Iniciado por defautl.

Finalizado: El proyecto se finalizó.

No Iniciado: El proyecto no se ha iniciado aún.

Iniciado: El proyecto ya está iniciado.

Suspendido: El proyecto fue suspendido.

Cotización: Se está en cotizaciones con el cliente para ver si el cliente lo aprueba.

Publicado: Si el check de selección se encuentra marcado indica que el proyecto va a aparecer en el portal del cliente.

Publicar asignación: Si el check de selección se encuentra marcado indica que la empresa dueña del proyecto va a poder ver a quién se le asignó las tareas.  Si no está marcado al cliente le aparece la tarea pero no le dice quién es el dueño.

Publicar fecha terminación: Si el check de selección se encuentra marcado indica que en el portal del cliente aparecerá la fecha en que se estima termine el proyecto.

Tamaño Máximo: Tamaño máximo de un archivo que se anexa al sistema.

Fecha Inicio: Fecha a partir de la cual arranca el proyecto.  Puede utilizar el calendario que ofrece el sistema.

Fecha Final: Fecha estimada para la conclusión del proyecto.

Tareas

El sistema muestra las tareas que están asignadas al proyecto:  El usuario puede Agregar, Eliminar, Copiar Agregar al Sitio del Proyecto, Eliminar del Sitio del Proyecto, Editar, Ver y Trasladar.

: Si el check de selección se encuentra marcado puede ejecutar la barra de iconos, tales como Editar. Copiar, etc.

Nombre: Nombre con el cual se identifica cada una de las tareas.

Prioridad: Prioridad que tiene la tarea dentro del proyecto.

Estado: Estado que tiene la tarea en el momento que se consulta la base de datos.

Completitud: El sistema maneja un grado porcentual para ver hasta que punto las tareas han sido ejecutadas. Cuando se han completado aparecen en color azul.

Fecha Entrega: Fecha indicada para que la tarea haya sido finalizada.

Asignado A: Código o nombre que identifica el miembro del grupo del proyecto al cual se le asignó la tarea.

Publicado: Si la tarea tiene indicado que si se publica le aparecerá en el portal del cliente.

Enviar: Este campo se utiliza para enviar la cotización al cliente cuando se encuentra lista.

Consultando una Tarea

Usted puede ver la información general de la tarea programada, tal como nombre, tarea predecesora, persona asignada, completitud, si se encuentra publicada, si se debe propagar el cambio o no.

Usted puede tener contenido enlazado ligado a la tarea, si ese encuentra publicado o no en el portal del cliente.

Historial de Asignaciones

Lo que dice esta sección del sistema es por cuántos miembros del equipo ha pasado la tarea y quién es el que la tiene actualmente.

Agregar una Tareas al Proyecto

En la siguiente pantalla Usted puede incluir todas las tareas que deben de llevarse a cabo para la ejecución del proyecto.

Nombre: Nombre completo de la tarea

Descripción: Descripción o nombre con el cual se va a identificar la tarea.

Asignado a: Nombre del miembro del grupo al que se le asigna la tarea del proyecto.

Tarea Predecesora: Cuál es la tarea predecesora para poder dar inicio a ésta.

Requerida: Si el check de selección se encuentra marcado indica que la tarea es indispensable ejecutarla para el proyecto.

Cobrable: Si el check de selección se encuentra marcado indica que la tarea debe de ser cobrada al cliente.

Estado: Estado en el que se encuentra la tarea. Cuando se está creando siempre por default esta en estado No Iniciado.

Finalizado por Cliente: La tarea se finalizó por parte del cliente.

Finalizado: La tarea ya se terminó.

No Iniciado: La tarea no se ha iniciado.

Iniciado: La tarea ya se inició.

Suspendido: La tarea fue suspendida.

Cotización: Se está preparando la cotización para que sea enviada al cliente.

Cotización Enviada: La cotización ha sido enviada al cliente.

Cotización Rechazada: El cliente no aprobó la cotización.

Cotización Aceptada: La cotización fue aprobada por parte del cliente.

Cobrar: Debe de generarse el cobro para la tarea asignada al proyecto.

Cobro enviado: Ya se envió el cobro al cliente por la tarea asignada al proyecto.

Procesando cobro: El cobro se está procesando para enviarlo al cliente.

Rechazada: Indica que la tarea fue rechazada.

El estado de una tarea puede ser modificado por el ADMIN, el Director de Proyecto o el miembro del equipo al que se le asignó la misma. Si la persona es un usuario solo tiene permiso de cambiar los campos estado y  comentarios.

Completitud: El sistema muestra el grado de avance que tiene la tarea.

Prioridad: Grado de prioridad que tiene la tarea.

Ninguna: La tarea debe hacerse pero no tiene prioridad.

Muy Baja: La tarea tiene una prioridad muy baja.

Baja: La tarea tiene prioridad baja para el proyecto.

Media: La tarea tiene prioridad Media para el proyecto.

Alta: La tarea tiene una prioridad ALTA para el proyecto por lo tanto debe ser ejecutada a la mayor brevedad.

Muy Alta: La tarea tiene una prioridad MUY ALTA para el proyecto por lo tanto debe ser ejecutada a la mayor brevedad, ésta por encima de todas las demás tareas.

Fecha Inicio: Toda tarea debe tener una fecha de inicio.

Fecha Final: Toda tarea debe tener una fecha de culminación Puede abrir el calendario con el siguiente botón.

Tiempo estimado: Tiempo en horas que se estima se lleve a cabo la tarea.

Tolerancia: Porcentaje de tiempo permitido para que se retrase la tarea, antes de que el sistema comience a indicar que la tarea está atrasada.

Tiempo Real: Tiempo que se necesitó para llevar a cabo la tarea.

Tarifa: El sistema maneja una tabla de cobro por hora en dólares para la realización de la tarea.

: Si el check de selección se encuentra marcado indica que no va a utilizar la tabla de precios, sino va a digitar un monto fijo.

Monto fijo en dólares por hora: Digite el monto que se cobra en dólares por hora para realizar la tarea.

Si el usuario utiliza monto fijo debe poner en horas debe estar en 1.

Comentarios predefinidos: Campo para indicar que el proyecto va a tener comentarios predefinidos.

No Aplica: No existen comentarios predefinidos.

Desarrollo: Existen comentarios predefinidos para el departamento de Desarrollo de Sistemas.

Análisis: Existen comentarios predefinidos para el departamento de Análisis de Sistemas.

Comentarios: Campo para anotar algún comentario referente a la tarea.

Severidad: Es recomendable categorizar por severidad todas las tareas del proyecto, para evaluar los grados de impacto sobre el mismo.

Bajo impacto: La tarea no repercute en forma directa en el proyecto.

Medio impacto: Cuando la tarea no es tan crítica por lo tanto el usuario la define como de medio impacto.

Alto impacto: Cuando la tarea es crítica por ejemplo si no se realiza detienen el proyecto.

Tipos de tarea: Toda tarea debe estar asignada a un tipo determinado.

Tareas de creación del PMI: El proyecto es la creación del PMI del proyecto.

TMS REQUEST: Es una tarea de requerimiento del T.M.S.

Reunión Interna: Es una tarea que se definió de la reunión interna que se dió en la empresa.

RequerImientos Socase: Indica que la tarea es un requerimiento solicitado por Socase, por ejemplo un log.

Requerimientos Partner:  Indica que la tarea es un requerimiento de un Partner.

Requerimientos Cliente: La tarea es un requerimiento específico del cliente, por ejemplo: agregar un campo informativo a una tabla en la base de datos.

Propagada: El resultado de la tarea debe documentarse dentro de la organización.

Normal de Proyecto: Es una necesidad del proyecto para mejorar el proyecto.

Next Release: Próxima versión que se estarán sacando para mejorar el producto.

Log de soporte: Es la corrección de un error del sistema.

Consultoría: Consultoría que se da al cliente para determinar una necesidad o no de un producto.

Capacitación: Consiste en  una capacitación para el uso del producto.

Administrativa: La tarea es del área administrativa.

Propagar cambio: El usuario puede propagar el cambio solamente cuando la tarea ha sido finalizada, es hasta ese momento que el sistema le pregunta si desea propagar ese cambio en el proyecto; si no a finalizado la tarea siempre aparece los guiones - - como se muestra en la pantalla.

Costos

En esta opción se ingresan todos los costos o gastos en los que se incurren para poder llevar a cabo un proyecto.

Agregando Costos a un Proyecto

Para agregar todos los costos en los que se ha invertido para llevar a cabo el proyecto solo debe seleccionar el botón AGREGAR y aparecerá la siguiente pantalla.

Descripción: Descripción completa del costo que genera el proyecto.

Unidades: Número de unidades que deben ser adquiridas para ejecutar el proyecto. Por ejemplo: se necesitan 5 proyectores para una demostración.

Costo Estándar: Costo estándar que tiene llevar a cabo el proyecto. Por ejemplo el costo para un proyectores es de $75 el alquiler por día.

Costo Real: Costo real que tiene la ejecución del proyecto. Por ejemplo nos hicieron una rebaja por que se alquilaron 5 proyectores a $60 por día.

Tarea: Seleccione el tipo de tarea que va a generar el costo. Este campo no es requerido por lo tanto puede dejarlo como N/A No Aplica.

Riesgos

En esta opción se ingresan todos los posibles riesgos que puede tener la empresa para que se no se lleve a cabo el proyecto.

Agregando Riesgos a un Proyecto

Para agregar riesgos a un proyecto solo debe seleccionar el botón AGREGAR y aparecerá la siguiente pantalla.

Descripción: Descripción completa del costo que genera el proyecto.

Probabilidad: Nivel de probabilidad de que el evento ocurra.

Nivel de impacto: Grado de impacto que tiene el riesgo dentro de la empresa si llegará a ocurrir.

Acción a ejecutar:  Defina el plan de acción que debe llevarse a cabo en caso de que el evento ocurra.

Plan B: Defina un segundo plan de acción que se debe ejecutar en caso de que el plan A no diera resultado.

Tarea: Indique el tipo de tarea asociada al riesgo.

Tickets

Esta opción contiene todas los temas de discusión que se abren durante la ejecución de un proyecto. Para cada uno el usuario puede Agregar, Eliminar, Cerrar, Agregar al Sitio del Proyecto, Quitar del Sitio del Proyecto o Ver el ticket.  Un ejemplo de un ticket es un error encontrado en el sistema.

Agregando Tickets al Proyecto

Para agregar tickets a un proyecto solo debe seleccionar el botón AGREGAR y aparecerá la siguiente pantalla.

Tema: Nombre o descripción breve del ticket o discusión.

Mensaje: Nivel de probabilidad de que el evento ocurra.

También puede para cada ticket agregar Contenido Enlazado.

Equipo

En esta opción se da mantenimiento al equipo de trabajo que se asigna para llevar a cabo el proyecto.  Igualmente se puede Agregar, Eliminar o Enviar el dato al portal del cliente.

Equipo: Es cualquiera persona que se ha definido como miembro del sistema, sea éste un miembro del cliente o un miembro interno de la compañía. Para cada proyecto se debe definir un Equipo de trabajo.

Agregando un Miembro al Equipo

Para agregar un miembro a un proyecto solo debe dar click en el botón AGREGAR y aparecerá la siguiente pantalla.

Si el check de selección se encuentra marcado indica que va a agregar a esa persona al equipo de trabajo para ese proyecto.

Nombre: Nombre del miembro del equipo de trabajo del proyecto.

Título: Título o grado académico que posee el miembro del equipo de trabajo.

Usuario: Nombre asignado al miembro para que ingrese a trabajar con el sistema TMS.

Teléfono trabajo: Número de teléfono de la oficina donde labora el miembro del equipo.

Publicado: Indica si se encuentra o no publicado en el perfil del cliente.

Horarios: Mediante esta opción Usted puede ver los días y horas de trabajo que tiene el miembro del equipo asignado para ese proyecto.

El sistema muestra la pantalla con las fechas de trabajo asignadas a los miembros del equipo.

Posiciónese en la fecha que desea consultar para ver el detalle de las tareas (como se muestra en la pantalla anterior)

Agenda: Mediante esta opción Usted puede ver la agenda de trabajo para el proyecto.

El sistema muestra la pantalla con las fechas de trabajo asignadas, también puede ver las tareas, el detalle del proyecto o enviar un mensaje al dueño del proyecto.

Posiciónese en los campos Hora o Tema para ver el detalle del evento (como se muestra en la pantalla anterior).

Posiciónese en los campos Tarea para ver el detalle del evento (como se muestra en la pantalla anterior).

Posiciónese en los campos Proyecto para ver el detalle del evento (como se muestra en la pantalla anterior).

Posiciónese en el campo Dueño para enviar un correo en Outlook al encargado directo del proyecto.

Contenido Enlazado

Esta opción es muy importante, todas los datos que se le hacen al cliente deben estar incluidas en el sistema. El contenido enlazado puede ser de un Proyecto, una Tarea o un Tickets. Esto nos permite tener acceso directo a la base de datos T.M.S  por lo tanto,  el usuario puede accesar internet y ver  la lo último se le hizo al cliente. En los casos en que un miembro del equipo asignado a un proyecto no se encuentre en la empresa y el cliente necesita que lo atiendan, entonces otro miembro pueda atender al cliente y pueda contar con toda la información necesaria. Los archivos que se suben deben de contener los acuerdos tomados con el cliente.  Estos archivos pueden haber sido generados en cualquier tipo de archivo, por ejemplo: Excel, Word, Texto, Gráfico, etc. Estos archivos son almacenados dentro del servidor de internet (WEB SERVER) y pueden ser consultados desde cualquier lugar con acceso a internet.

Agregando Contenido Enlazado al Proyecto

Para agregar dele un click en el botón AGREGAR y aparecerá la siguiente pantalla.

Subir  Sube el documento que tengo en el disco duro y se guarda en el repositorio de la base de datos del T.M.S. Ejemplo: Usted elaboró una oferta en Microsoft Word para posteriormente agregarlo al sistema como contenido enlazado ya que de esa forma queda guardado en el sistema.  

Examinar: Utilice el botón Examinar para seleccionar la ruta de acceso a los archivos.

Comentarios:  Campo para anotar alguna observación o comentario que se estime conveniente pera el contenido enlazado.
